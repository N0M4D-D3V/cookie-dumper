What it does

Cookie Dumper watches a custom list of hostnames and can dump their cookies either to disk or to an HTTP endpoint. The popup behaves like a tiny SPA: manage domains on the home view, tweak delivery settings on the Advanced screen, and trigger manual dumps without leaving the extension.

Human machine interface

Home

  • Manage Watched domains: add manually or tap the bookmark button to pull the active tab's hostname.
  • Each row can be removed, and clicking a domain opens the inline editor.
  • The status console lives here and types feedback for all actions.

Domain editor

  • Rename or add a hostname inline.
  • The back button returns to home and refreshes it.

Advanced settings

  • Toggle AUTO-DUMP to watch navigation events in background.
  • Choose a delivery mode (File or HTTP) and save/clear the HTTP endpoint.
  • Manual actions now live here: Using the delivery mode chosen, DUMP TAB exports the active tab; DUMP ALL exports every watched domain.

Dump modes

File

  • Stores a JSON payload under cookie-dumps/<timestamp>.json.
  • Uses Chrome downloads; no prompt thanks to the predefined path.

HTTP

  • POSTs the JSON payload to your configured endpoint.
  • Requires a non-empty endpoint.

Mode defaults to disabled. Pick File or HTTP in Advanced settings to allow auto/ manual dumps to run.

Automation

  • A background service listens to tab navigation completion.
  • When AUTO-DUMP is on and a delivery mode is set, watched domains (and their subdomains) trigger auto-dump.
  • The dump process groups cookies per site and sends them in one payload; use it on demand from Advanced settings.

Data handling & encryption

  • Cookies are grouped by site before serialization.
  • Payloads are double-encrypted and decrypted just-in-time for the final JSON.
  • Extension storage keeps only configuration: watched domains, AUTO-DUMP flag, delivery mode, and HTTP endpoint.

Permissions

  • cookies to read cookies for watched hosts.
  • tabs to inspect the active tab URL and navigation.
  • downloads to write JSON files in File mode.
  • storage for domain lists and settings.
